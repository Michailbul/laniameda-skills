# Routing Quick Reference

When Michael says something, route it here:

| What Michael says | File to update |
|---|---|
| Studio name, stage, who we are, founder context | `company/identity.md` |
| North star, goals, what we're building toward, phase arc | `company/vision.md` |
| Core values, working principles, how we behave | `company/values.md` |
| The big positioning statement, why the studio exists | `company/manifesto.md` |
| Services offered, what we build for clients | `agency/what-we-do.md` |
| How we work, process, collaboration approach | `agency/how-we-work.md` |
| Client types, who's right for us, who isn't | `agency/clients.md` |
| Colors, typography, dark/light mode, shape language, motion | `brand/visual.md` |
| Brand voice, tone, what we say or don't say | `brand/voice.md` |
| Content formats, quality bar, what gets killed | `brand/content-standards.md` |
| Content pillars, Marketing First, content engine | `departments/marketing.md` |
| Tech decisions, stack, tools, infrastructure | `departments/engineering.md` |
| AI models in use, prompt vault rules, creative tools | `departments/creative.md` |
| Project status update, new project, repo change | `projects/index.md` |
| Agent domain, role, rule, protocol | `agents/roster.md` |

## Common Multi-File Updates

**"We don't post generic AI content"**
→ `brand/voice.md` (What We Don't Post About) + `brand/content-standards.md` (What Gets Killed)

**"Add a new content pillar"**
→ `departments/marketing.md` (Content Pillars table) + `brand/content-standards.md` (Content Pillar Filter table)

**"New AI model we're using"**
→ `departments/creative.md` (AI Tools in Active Use table)

**"New project started"**
→ `projects/index.md` (Active Projects table + add detail section)

**"Changed how we think about clients"**
→ `agency/clients.md` + possibly `company/manifesto.md` if it's a major positioning shift
