# Brand Identity & Voice

## Brand Name

- **Wordmark**: `laniameda` — lowercase, display serif (Instrument Serif)
- **Product suffix**: italic, 80% opacity (e.g., *Gallery*, *Hub*)
- **Logo mark**: ink-colored diamond (rotated 45deg square), 10x10px

---

## Visual Atmosphere

### Mood Keywords

`Warm editorial` · `Studio console` · `Paper & ink` · `Volcanic warmth` · `Artisan digital` · `Cinematic gallery`

### Description

The visual language draws from **printed art catalogues** and **film photography studios**. Surfaces feel like sun-warmed archival paper. Text reads like ink on cotton stock. Accents glow like embers — coral, amber, warm red — never neon, never cold.

The UI recedes to let content be the hero. Chrome is minimal. Interactions are tactile but restrained — a subtle lift here, a warm glow there. Nothing bounces, nothing flashes.

### Emotional Qualities

| Quality | Expression |
|---------|------------|
| Warmth | Paper-toned backgrounds, amber/coral accents, warm gradients |
| Confidence | Strong monospace labels, uppercase tracking, ink-dark text |
| Craft | Grain texture, letterpress shadows, editorial type hierarchy |
| Clarity | Generous whitespace, strict hierarchy, content-first layout |
| Technical precision | Monospace data, structured grids, code-native patterns |

## Brand Voice

### Tone

**Quiet confidence** — authoritative but never loud. Like a senior engineer who doesn't need to prove anything.

### Register

For developer-facing products: sharp, technical, direct, no fluff. Written for people who live in the terminal and read docs at 11pm.

### Test

> "These people get it."
>
> If a developer reads the hero at 11pm after a long coding session and thinks that — it works.

### Voice Rules

| Do | Don't |
|----|-------|
| Be direct and technical | Use SaaS corporate speak |
| Assume intelligence | Over-explain obvious things |
| Use terminal metaphors | Use "game-changing" or "unlock potential" |
| Write tight, punchy copy | Write walls of marketing fluff |
| Let the product speak | Over-promise or use superlatives |
| Reference real workflows | Speak in abstract benefits |

### Copy Examples

**Good:**
- "Your agents are only as good as their context."
- "One command. That's it."
- "Self-hosted editor for the files that make your AI agents smart."

**Bad:**
- "Unlock the full potential of your AI workflow!"
- "The game-changing platform for next-gen AI agents"
- "Supercharge your productivity with our innovative solution"

## Audience

**Primary**: AI engineers, Claude Code users, Codex users, vibe coders
**Secondary**: AI PMs, developer tool enthusiasts, open source contributors

These people:
- Run multiple AI agents daily
- Care about context quality (SOUL.md, skills, memory)
- Prefer CLI-first tools
- Value MIT license, self-hosted, no vendor lock
- Read code more than marketing pages

---

## Design Principles

1. **Content-first** — UI recedes, content is the hero
2. **Warm, not cold** — every surface carries warmth, no clinical whites
3. **Editorial + Industrial** — soft paper meets sharp monospace
4. **Restrained motion** — Apple/Tesla feel: smooth, inevitable, barely noticed
5. **Light only** — the design system is light-mode. Dark elements (terminals, inverse panels) are accent moments within a warm canvas
6. **Generous whitespace** — editorial layouts breathe, never cramped
